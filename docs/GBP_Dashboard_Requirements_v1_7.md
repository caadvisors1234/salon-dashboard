## 1. プロジェクト概要

### 1.1 背景と目的

代理店・コンサルティング会社が、クライアントのGoogle Business Profile（GBP）パフォーマンスおよびHot Pepper Beauty（HPB）のサロンパフォーマンスを一元管理し、データに基づいた施策提案を行うためのWebダッシュボードを構築する。

クライアント向けには自社データの閲覧機能を提供し、レポートのPDF出力にも対応する。

### 1.2 スコープ

| 項目 | 内容 |
|---|---|
| アプリ種別 | クローズドWebアプリケーション（招待制） |
| 利用者 | 代理店・コンサルタント + クライアント |
| 管理対象 | 初期: 100店舗規模で設計（将来的に500店舗まで拡張可能な設計） |
| ユーザー数 | 初期: 数十名規模（管理者・担当者・クライアント合計） |
| 対応デバイス | PCブラウザ + スマホレスポンシブ |
| UI言語 | 日本語のみ |
| データソース | Google Business Profile API + Hot Pepper Beauty（CSV手動アップロード） |

## 2. ユーザー・権限設計

### 2.1 ロール定義

| ロール | アクセス範囲 | 主な操作 |
|---|---|---|
| 管理者（Admin） | 全クライアント・全機能 | ユーザー管理、クライアント管理、全データ閲覧、レポート出力、システム設定、HPB CSVアップロード |
| 担当者（Staff） | 担当クライアントのみ | 担当クライアントのデータ閲覧、レポート出力、HPB CSVアップロード |
| クライアント（Client） | 自社データのみ | 自社ビジネスプロフィールのデータ閲覧、レポートダウンロード |

> **注記:** HPB CSVのアップロード権限は Admin / Staff に限定する。Client ロールは閲覧のみ。

### 2.2 認証方式

- Supabase Authを使用したメール + パスワード認証
- クローズドアプリ: 管理者がユーザーを招待・登録（セルフサインアップ不可）
- パスワードリセット機能（メールリンク方式）
- JWTトークンによるセッション管理

## 3. データ取得仕様

### 3.1 データソース一覧

#### 3.1.1 GBP（API自動取得）

| API | 取得データ | 取得粒度 | 備考 |
|---|---|---|---|
| Business Profile Performance API v1 | インプレッション、アクション指標 | 日次 | `fetchMultiDailyMetricsTimeSeries` を使用 |
| Business Profile Performance API v1 | 検索キーワード指標（value/threshold） | 月次 | `searchkeywords.impressions.monthly.list` を使用 |
| GBP API v4.9（Reviews） | 平均評価（`averageRating`）、レビュー総数（`totalReviewCount`）、レビュー一覧 | 日次スナップショット（都度取得） | `accounts.locations.reviews.list` を使用。OAuth 2.0認証。**無料** |

> **注記（v1.7 確定方針）:** 評価・レビュー数の取得は **GBP API v4.9（Reviews）に統一**する。Places API（New）は本リリースで使用しない。
>
> **選定理由:**
> - GBP APIアクセス申請が承認されると、標準クオータが付与される
> - GBP API v4.9 Reviews は無料で利用可能
> - `averageRating` / `totalReviewCount` を同時取得できる
> - レビュー返信API（将来拡張）に対応可能
>
> **参考: GBP API v4.9（Reviews）と Places API（New）の比較**
>
> | 比較項目 | GBP API v4.9（Reviews） | Places API（New） |
> |---|---|---|
> | 総合評価 | ✅ `averageRating` | ✅ `rating`（Enterprise SKU） |
> | レビュー総数 | ✅ `totalReviewCount` | ✅ `userRatingCount`（Enterprise SKU） |
> | 個別レビュー一覧 | ✅ ページネーションで全件取得可 | △ 最大5件（`reviews`、Enterprise + Atmosphere SKU） |
> | レビュー返信機能 | ✅ あり | ❌ なし |
> | 認証方式 | OAuth 2.0 | APIキー |
> | 課金 | 無料 | 従量課金（SKUベース） |
>
> **v1.0スコープ:** レビュー全件閲覧・返信UIは実装しない。`averageRating` / `totalReviewCount` のスナップショット取得に限定する。
>
> **フォールバック:** v4.9 Reviews が将来廃止された場合は Places API（New）へ切替える。API呼び出しは抽象化レイヤーで分離する（セクション9.2参照）。

#### 3.1.2 Hot Pepper Beauty（CSV手動アップロード）

| データソース | 取得データ | 取得粒度 | 取り込み方法 |
|---|---|---|---|
| サロンボード「レポート - 基本情報」CSV | サロン情報PV数、CVR、ACR 等（自店値 + 同P同Aエリア平均） | 月次 | 店舗ごとにCSVを手動ダウンロード → ダッシュボードからアップロード |

**CSVファイル仕様（サロンボード出力形式）**

| 項目 | 詳細 |
|---|---|
| ファイル形式 | CSV（Shift_JIS エンコーディング） |
| ファイル名パターン | `HPBレポート-基本情報_YYYYMMDDHHmmss.csv` |
| データ構造 | 1行目: ヘッダー、2行目以降: 月次データ（1行=1ヶ月分） |
| 対象年月カラム | `対象年月`（YYYYMM形式） |
| 指標ペア構造 | 各指標は「自店」と「同P同A平均（同プラン同エリア平均）」の2列で構成 |
| 全カラム数 | 21列（対象年月1列 + 指標ペア20列。重複カラムを含む） |

#### 3.2.3 検索キーワード指標

| 指標名 | エンドポイント | レスポンス構造 | 制約 |
|---|---|---|---|
| 検索キーワード指標 | `searchkeywords.impressions.monthly.list` | `insightsValue.value` または `insightsValue.threshold` | 月次集計のみ。更新遅延の明確なSLAは非公開 |

> **実装ルール（v1.7確定）**
> - `value` が返る場合: 実数として表示・集計する
> - `threshold` が返る場合: `"<threshold"` で表示する（例: `<15`）
> - 前月比は **当月・前月ともに `value` の場合のみ** 算出する
> - いずれかが `threshold` の場合、前月比は `算出不可` と表示する

### 3.3 データ収集・取り込み設計

#### 3.3.1 GBPデータ（自動バッチ）

- **日次バッチ（毎日 AM 3:00 JST）**
  - パフォーマンス指標（日次）
  - 評価・レビュー数（日次スナップショット）
- **月次バッチ（毎月8日 AM 3:00 JST）**
  - 検索キーワード（月次）
- **失敗時リトライ**
  - 最大3回、指数バックオフ
- **自動バックフィル**
  - 直近30日を再取得対象として欠損日を埋める
  - 30日を超える欠損はアラートを発行し、管理者の手動再実行対象とする

> **注記（時刻選定）:** AM 3:00 JST は運用上の混雑回避を目的とした設定であり、Google側更新完了時刻を保証するものではない。

#### 3.3.2 HPBデータ（CSV手動アップロード）

**アップロードフロー**

1. Admin / Staff がサロンボードからCSVを手動ダウンロード
2. ダッシュボード「HPBデータアップロード」画面で対象店舗を選択
3. CSVをアップロード
4. サーバー側でバリデーション・パースを実行
5. 取り込み件数 / エラー / ワーニングを表示

**バリデーションルール**

| チェック項目 | 内容 | エラー時の挙動 |
|---|---|---|
| エンコーディング | Shift_JISであること | エラー表示（UTF-8等は非対応） |
| ヘッダー検証 | 必須カラムが存在すること | 不足カラム名を明示してエラー |
| 重複カラム名検知 | 同一カラム名が複数存在 | ワーニング表示。先頭出現列を採用 |
| 対象年月 | YYYYMM 6桁数値 | 該当行をスキップしワーニング |
| 数値型（整数） | PV系・予約売上高は整数のみ許可 | 該当行をスキップしワーニング |
| 数値型（小数） | CVR/ACR/予約数は小数を許可 | 該当行をスキップしワーニング |
| CVR/ACR値範囲 | 0〜100（百分率） | 範囲外はワーニング表示で取り込み継続 |
| 重複レコード | 同一店舗・同一年月が既存 | 上書き確認後にUPSERT |

**CSVパース処理方針**

- 全カラムをDBへ取り込む（UI表示は段階拡張）
- カラム名ベースのパースを優先
- `サロン情報PV数` 重複は先頭列を正とする
- 重複検知時はワーニングログを残す
- カラム名ベース失敗時のため、列位置マッピングをフォールバックとして保持

#### 3.3.3 データ保持・コンプライアンス方針

| データ種別 | 保持期間 | 方針 |
|---|---|---|
| GBP集計データ（`daily_metrics`, `monthly_keywords`, `rating_snapshots`） | 365日 | ダッシュボード分析用に保持 |
| レビュー本文・投稿者情報 | 保持しない（DB永続化なし） | v1.0では保存対象外。必要時は都度取得 |
| バッチログ | 365日 | 障害調査・監査用 |
| HPBアップロードログ | 365日 | 運用監査用 |
| HPB CSV原本（Storage） | 365日 | 再取込と監査用途 |

> **注記:** API利用規約の変更に備え、保持期間は設定値で短縮可能とする。

#### G. HPB KPIサマリーカード

| カード | 自店値 | エリア平均 | 表示形式 |
|---|---|---|---|
| サロン情報PV数 | 当月PV + 前月比 | 同P同A平均PV | 整数 |
| CVR | 当月CVR + 前月比 | 同P同A平均CVR | %（小数1位） |
| ACR | 当月ACR + 前月比 | 同P同A平均ACR | %（小数1位） |

#### H. HPB指標推移グラフ

| グラフ | Y軸 | 系列 |
|---|---|---|
| サロン情報PV数推移 | PV数 | 自店（実線）/ エリア平均（破線） |
| CVR推移 | CVR（%） | 自店（実線）/ エリア平均（破線） |
| ACR推移 | ACR（%） | 自店（実線）/ エリア平均（破線） |

- 期間: 保有全月
- X軸: 月
- レイアウト: PVフル幅、CVR/ACRは2カラム（モバイル時縦積み）

## 6. レポート出力仕様

### 6.1 PDFレポート

- 出力単位: 店舗単位 / クライアント単位
- 期間指定: 任意の月範囲
- 含有内容: GBP KPI + グラフ + キーワードテーブル + HPBセクション（データあり時）
- ヘッダー: 代理店ロゴ + クライアント名
- ロゴ管理: 代理店ロゴ1点を静的ファイルで配置（管理画面変更なし）
- 生成方法: ダウンロードボタン押下で生成
- 同時生成制限: キュー処理 + 最大2件（初期設定）

> **注記:** HPBデータ未登録店舗はHPBセクションを省略して生成する。

## 9. 前提条件・リスク

### 9.1 前提条件

- GBP APIアクセス申請が承認されること（審査目安: 14日以内）
- クライアントがGBP管理権限を代理店へ付与すること
- 既存VPSが利用可能であること
- HPB CSVを店舗側で月次出力できること

### 9.2 リスク

| リスク | 影響度 | 対策 |
|---|---|---|
| GBP APIアクセス申請却下 | 高 | 早期申請、審査期間を考慮して前倒し着手 |
| Google APIレートリミット | 中 | 300 QPM前提で分散実行、キャッシュ、指数バックオフ |
| 検索キーワードのthreshold返却 | 中 | `<threshold` 表示、前月比は算出不可に統一 |
| 検索キーワード更新遅延 | 低 | 月次仕様としてUI/運用で周知 |
| v4.9 Reviewsの将来廃止 | 中 | 抽象化レイヤー + Places API切替手順を保持 |
| HPB CSVフォーマット変更 | 中 | ヘッダー検証、重複カラム検知、ワーニング運用 |
| HPBアップロード漏れ | 低 | 管理画面アラート + メール通知 |
| バッチコンテナ停止 | 中 | 再起動ポリシー + 死活監視 + 30日バックフィル |
| PDF生成負荷増大 | 中 | 同時実行上限、必要時に専用ワーカー化 |
